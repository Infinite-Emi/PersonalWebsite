Project: Astronomical Image Classification

The Problem

Manual classification of galaxies is a huge bottleneck for astronomical research. This project aimed to automate the process by training a deep learning model to classify galaxies from image data, inspired by the Galaxy Zoo citizen science project.

The Action

I built and trained a series of Convolutional Neural Networks (CNNs) using Python, TensorFlow, and Keras. The process involved starting with benchmark datasets (MNIST, CIFAR-10) before tackling the complex, 10-class Galaxy10 dataset. A key part of the process was implementing data augmentation (flips, rotations) to improve model generalization.

The Result

The final model, using a DenseNet121 architecture, achieved 80% test accuracy. This successfully demonstrated the viability of automating this complex visual task. The chart below shows the significant class imbalance in the training data—a key challenge that had to be overcome.

Project: Tree Species Recognition

The Problem

Real-world datasets are often imbalanced, leading to biased AI models. This project was a formal investigation into this challenge, using aerial imagery of German forests to test strategies for improving model performance on rare tree species.

The Action

I designed a structured experiment, creating five distinct CNN models to systematically test the impact of different techniques: reducing classification granularity, adding data augmentation, applying proportional class weighting, and increasing model depth. This isolated the effect of each strategy.

The Result

The experiments highlighted a critical trade-off. While class weighting dramatically improved the model's ability to identify rare species (recall), it harmed its overall precision. The chart below visualizes the precision vs. recall for the different experimental models, demonstrating this trade-off in action.

Project: Electronic Sales Analysis

The Goal

To conduct an exploratory data analysis on a synthetic electronic sales dataset. The objective was to clean the data, analyze it from multiple perspectives (customer demographics, product performance, sales trends), and visualize the findings to uncover potential business insights.

The Process

The entire analysis was performed in a Python Jupyter Notebook using the Pandas library for data manipulation and Matplotlib/Seaborn for visualization. The process included initial data sanity checks, cleaning (e.g., handling null values), feature engineering (e.g., extracting month and weekday from dates), grouping and aggregation to calculate key metrics, and creating a suite of plots to communicate the results.

Key Insights

The analysis revealed several interesting patterns. Smartphones were the top-selling product by both quantity and revenue. There was a significant jump in monthly revenue early in the year. Interestingly, while nearly half of all customers were repeat buyers, their average spend was only slightly higher than one-time customers.
